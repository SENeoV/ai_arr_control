# Contributing

Thank you for considering contributing to AI Arr Control. Please follow these guidelines to make meaningful, reviewable contributions.

Basic workflow
- Fork the repository and create a branch per feature or bugfix.
- Write tests for new features or bug fixes.
- Follow code style: Black formatting and ruff linting.
- Open a PR describing the change, motivation, and any backward compatibility impact.

Testing
- Install dev dependencies: `pip install -e .[dev]`
- Run tests: `pytest`

Code style
- Use `black` for formatting and `ruff` for linting.

License and CLA
- By contributing you agree your changes will be licensed under the project license.
